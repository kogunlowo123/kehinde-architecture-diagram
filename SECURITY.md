# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email: **kogunlowo@gmail.com**

Include the following in your report:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

You will receive an acknowledgment within 48 hours and a detailed response within 7 days.

## Security Best Practices

This project follows these security practices:

- No sensitive data is embedded in HTML or client-side code
- External resources are loaded over HTTPS
- Content Security Policy headers are recommended for deployment
- User input is sanitized to prevent XSS attacks
- Third-party scripts and dependencies are reviewed before inclusion

## Disclosure Policy

We follow a coordinated disclosure process. Please allow us reasonable time to address any reported vulnerabilities before public disclosure.
