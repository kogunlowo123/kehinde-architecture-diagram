# Contributing to Kehinde Architecture Diagram

Thank you for your interest in contributing! This guide will help you get started.

## How to Contribute

1. **Fork** the repository
2. **Create a branch** for your feature or fix (`git checkout -b feature/my-feature`)
3. **Make your changes** and ensure they follow the project conventions
4. **Test** your changes thoroughly
5. **Commit** with a clear, descriptive message
6. **Push** to your fork and open a **Pull Request**

## Development Setup

```bash
# Clone your fork
git clone https://github.com/<your-username>/kehinde-architecture-diagram.git
cd kehinde-architecture-diagram

# Open index.html in your browser to preview changes
```

## Code Standards

- **Valid HTML**: Ensure all HTML is valid and well-formed
- **Responsive Design**: Pages should work across desktop, tablet, and mobile viewports
- **Accessibility**: Follow WCAG guidelines for accessible markup
- **Semantic HTML**: Use appropriate semantic elements (`<header>`, `<nav>`, `<main>`, `<section>`, etc.)
- **CSS**: Keep styles organized and avoid inline styles where possible

## Pull Request Guidelines

- Keep PRs focused on a single change
- Test your changes across multiple browsers and screen sizes
- Update documentation if needed
- Describe what your PR does and why

## Reporting Issues

- Use GitHub Issues to report bugs or request features
- Include steps to reproduce any bugs
- For security vulnerabilities, see [SECURITY.md](SECURITY.md)

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.
